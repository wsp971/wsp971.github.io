两个问题

发表于 2016-06-30   |   分类于 生活   |     |   阅读次数

今天遇到了两个问题，最后终于也解决了。在此记录一下：

  • 自己的windows笔记本在公司上不了网，公司只允许用mac笔记本或者手机来上公共wifi,故思来想去，只能用修改ua 来伪装手机来上网，重要的是我以后可以背着自己的本本来公司上班了。之前，只能离线玩。没网什么都干不了。
  • 下午在搭建新的项目运行环境，由于公司电脑在内网下下载不了node 依赖项，故只能采用，在自己电脑上下载好，copy到公司电脑。在copy过程中出现了node依赖模块目录太长而无法copy过长的目录，经过一番搜索，终于也解决之。

windows 下修改 user-agent 来伪装手机浏览器 上网

原理想清楚很简单，每个浏览器在发送http请求的时候都会附带本身的信息给服务器，这个默认情况下，用户不用管，程序员不用管，浏览器会在发送请求时自动附带信息，这样服务器就可以拿到客户端浏览器的信息了，包括操作系统，版本，内核等等的信息。而这些信息，在浏览器里就是 user-agent，简称ua ,公司的上网限制，也只做了限制windows pc电脑，故有了思路，我们修改下浏览器的 ua,即可骗过服务器。于是网上搜了下相关手机的ua，替换了上去即可，chrome修改最为方便，搜了一个ua如下：

"Mozlila/5.0 (iphone; CPU iphone OS 8_0 like Mac OS X) AppleWebKit/600.1.3(KHTML,like Geckko) Version/8.0"

chrome 下设置步骤如下：

  • 右击chrome 属性，在目标栏目里添加：

-user-agent='Mozlila/5.0 (iphone; CPU iphone OS 8_0 like Mac OS X) AppleWebKit/600.1.3(KHTML,like Geckko) Version/8.0'

其中 user-agent的值就是搜到的ua,包括mac电脑，及各种浏览器的的ua。

用ROBOCOPY来解决window下 文件路径太长而不能复制的问题

一般在复制文件，路径过长时，就会弹出如下错误框：

网上有人搜到微软的官方解释,链接如下：

https://msdn.microsoft.com/en-us/library/windows/desktop/aa365247
其中有一句话：

The shell and the file system have different requirements. It is possible to create a path with the Windows API that the shell user interface is not able to interpret properly.

意思就是说，微软平台下，应用程序可以超出常规的260字符限制而达到32767这样长度的路径，我们手动copy不了，那么就只能用应用程序自copy了。
