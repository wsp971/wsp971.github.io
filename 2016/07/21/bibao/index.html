作用域，执行上下文，闭包

发表于 2016-07-21   |   分类于 技术   |     |   阅读次数

javascript中最难理解和掌握的几个点，今天来总结一下：

作用域

作用域，编程语言里即指变量的作用域，通俗一点讲就是变量的作用范围，javascript 没有“块”级作用域，所谓“块”，就是指“{}” 括起来的部分。例如：if,for等语句大括号内的内容：

var i=10;
if(i>5){
    var j=i;
}
console.log(j);// 10

可以看到在if之外，还可以访问到括号里边定义的变量j;

我们要记住的是：javascript中出了全局作用域，只有函数可以创建作用域，全局作用域，即是我们js所执行的最外层执行环境，例如浏览器script标签提供的执行作用域。

作用域类似于变量所定义的地盘一样，可以隔离变量的作用范围。

var a=10; //全局作用域
function f1(){
    // 函数f定义的作用域
    var b=9;
    var c=8;
    function g1(){
        //函数g定义的作用域
        var d=7;
        var e=6;
        console.log(a);//10
        console.log(b);//9
        console.log(c);//8
        console.log(d);//7
        console.log(e);//6
        console.log(f);//f is undefined
        console.log(g);//g is undefined
    }
    function e1(){
        var f=5;
        var g=4;
        var a="a",b="b",c="c",d="d",e="e";
        g1(); 
    }
    e1();
}
f1();

看起来很简单，很容易理解，作用域是有层级关系的，函数g1作用域内 自然可以访问函数f1作用域内的变量，即使我们在调用的e1函数内，我们队这些变量重新赋值，在执行过程中取值时，对于函数自己作用域内没有定义的变量，他会从父级作用域来寻找这个变量，直到全局作用域，找不到，就会报错，这种在函数自身内没有定义的变量，我们称之为自由变量，这个从函数定义的作用域往上级找变量的顺序，我们称之为作用域链
