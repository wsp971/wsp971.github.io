浏览器对象模型（browser Object model）

发表于 2016-08-23   |   分类于 技术   |     |   阅读次数

什么是BOM

由于现代浏览器实现了javascript 交互方面的相同方法和属性，但浏览器对象没有正式标准

Window 对象

所有浏览器都支持window 对象，表示浏览器窗口，所有javascript 全局对象，变量 函数，均为window对象的成员

html 的 dom 根节点 document 对象也是 window 的成员 即 window.document===document

window 的尺寸（浏览器的视图窗口，不包含工具栏和滚动条）

对于 Internet Explorer、Chrome、Firefox、Opera 以及 Safari 浏览器

  • window.innerHeight - 浏览器窗口的内部高度
  • window.innerWidth - 浏览器窗口的内部宽度

对于 Internet Explorer 8

document.documentElement.clientHeight
document.documentElement.clientWidth

兼容写法

var w=window.innerWidth
|| document.documentElement.clientWidth
|| document.body.clientWidth;

var h=window.innerHeight
|| document.documentElement.clientHeight
|| document.body.clientHeight;

document.body.clientHeight||clientWidth 在现代浏览器中表示的是 body 的宽度和高度

window.screen 对象

包含用户屏幕的信息

screen.availWidth // 屏幕可用宽度
screen.availHeight //屏幕 可用高度

Navigate对象

navigate对象包含了浏览器和浏览器所在桌面的相关信息。主要有以下属性：

属性 解释
appName web浏览器全称。 如：“microsoft Internet Explorer” “Netscape”
appVersion 数字开头，包含浏览器厂商和版本信息
appCodeName 浏览器先关的内部代码名，都为 Mozilla
userAgent 浏览器在她的 USER-AGENT HTTP头部中发送的字符串
platform 浏览器所在的操作系统
onLine 匹配任何非空白字符。
cookieEnabled cookie 是否可用
plugins 浏览器插件集合
language 浏览器语言，ie不支持
userLanguage 浏览器首选语言,IE支持 为“zh-cn”
systemLanguage 系统语言,IE支持 为“zh-cn”

cookies

cookie 是存储于访问者计算机中的变量，当计算机通过浏览器请求某个页面时，就会附带着cookie过来到服务器。通过js 可以访问，并且设定cookie 的值，cookie在浏览器下是下挂在document对象下的属性。因为浏览器本身没有提供获取或者设置单个cookie的方法，故操作cookie都要写代码操作。一般用特定分隔符分割的字符串来代替。

cookie的属性

cookie 的属性除了可以通过 name/value 设置外，还可以设置有效期，作用域，安全性等。

  • expires/max-age属性

cookie的默认有效期很短暂，它只能持续在web浏览器的会话期间，一旦用户关闭浏览器，cookie保存的数据就丢失了。如果要延长，那么就通过设置expires属性，现在max-age属性已经替代了expires属性，max-age用秒来设置有效期。

max-age一般设置为正数，表示该cookie在max-age秒后自动失效，浏览器会将正数的cookie持久化。即写到对应的cookie文件中。无论关闭浏览器还是关闭电脑，cookie仍然有效；

如果cookie为负值，则表示该cookie只在本浏览器窗口及本窗口打开的子窗口有效。关闭窗口后即失效。max-age为负数的cookie为临时性cookie,不会被持久化，不会被写入到cookie文件中。cookie的默认max-age为-1

如果max-age设置为0，则表示删除该cookie。cookie机制没有提供删除cookie的方法。因此通过设置该cookie的值为0 来实现即时删除cookie的效果。失效的cookie会被浏览器从cookie文件或者内存中删除。

  • domain属性

domain属性可以使得多个web服务器共享cookie。domain属性的默认值是创建cookie的网页所在的服务器的主机名。不能将一个cookie的域设置成服务器所在的域之外的域。
例如，让位于a.wsp.com的服务器能够读到b.wsp.com的设置的cookie的值，如果b.wsp.com的页面创建的cookie的path属性设置为/ ，把domain属性设置成.wsp.com,那么所有位于b.wsp.com的网页和所有a.wsp.com的网页以及wsp.com域的其他服务器上的网页都可以访问这个cookie。

  • path属性

path指定与cookie关联在一起的网页，在默认情况下，cookie与创建它的网页有关，并对该网页和该网页同目录后者子目录的其他web页面有关
，比如： http://wangshiping.net/categories/a.html 下创建的cookie 在 http://wangshiping.net/categories/b.html 下可以访问，但是在 http://wangshiping.net/guitar/index.html 下是不可以访问的。如果要设置可访问，则要设置path。例如：
http://wangshiping.net/categories/test/a.html 下设置的cookie 要在 http://wangshiping.net/categories/b.html 下可以访问。那么就要设置cookie的path为/categories。当然，如果要设置站点下的所有cookie都可以互相访问，则设置path 为：/ 即可。将cookie 的path设定为“/”,即 cookie 拥有了和localstorage 相同的作用域。它在该站定请求的任何一个页面都会上送该cookie。

  • secure属性

secure属性是一个布尔值，用来制定如何在网络上传输cookie，cookie默认是以不安全的形式（普通的 HTTP 链接）传递的。而一旦cookie被标识为 “安全的”，那就只能浏览器和服务器通过https或者其他安全协议连接的时候才能够传递它。

  • httpOnly属性

httpOnly 属性限制了cookie 对HTTP请求的作用范围。特别的。该属性指示用户代理忽略那些通过非“HTTP”方式对cookie的访问（比如浏览器暴露给js的接口）。注意：httpOnly属性和Secure 属性相互独立。

cookie的编码及解码

由于cookie的键值对不用需包含分号，逗号和空格符，任何被存储为cookie的值都应该被编码，前端一般用 encodeURIComponent 编码 和 decodeURIComponent 解码。

同名的cookie

同名的cookie,不同的domain,或不同 path 属于不同的cookie； 同名的cookie,相同的domain,相同的path,属于同一个cookie

cookie的局限性

前面说了浏览器在发送请求时，会自动发送cookie,所以cookie的存储不能太大，而且RFC 2956 标准不用需浏览器保存超过300个 cookie,为每个web服务器保存的cookie数不能超过20个，而且每个cookie保存的数据不超过4kb;当然现在有浏览器的允许cookie的总数超过300个。总之现在不建议cookie作为客户端存储的方案。现在可有的客户端存储方案有 html5的localStroage 和 ie的 userData 等。

####实现一个基于cookie 的类localStorage 的存储方案

//实现一个基于cookie的少量数据存储api
 function cookieStorage(maxage,path){
       //把cookie按特定格式分割后以key-value形式保存在对象中
       var cookie=(function(){
            var cookies={};
            var all=document.cookie;
            if(all===""){
                return cookies;
            }
            var list=all.split(";");
            for(var i=0;i<list.length;i++){
                var cookie=list[i];
                var p=cookie.indexOf("=");
                var name=cookie.substring(0,p);
                var value=cookie.substring(p+1);
                value=decodeURIComponent(value);
                cookies[name]=value;
            }
            return cookies;
        })();
       //保存cookie 的key值
       var keys=[];
       for(var key in cookie){keys.push(key);}
       //cookie 的长度
       this.length=keys.length;
       //提供api 根据位置返回cookie值
       this.key=function(n){
            if(n<0||n>keys.length){return null;}
            return keys[n];
       } 
       //根据cookie的key，返回value
       this.getItem=function(name){
            return cookie[name]||null;
       }
       //以key,value值设置cookie
       this.setItem=function(key,value){
            if (!(key in cookie)){
                keys.push(key);
                this.length++;
            }
            cookie[key]=value;
            var this_cookie=key+"="+encodeURIComponent(value);
            if(maxage) this_cookie+=";max-age="+maxage;
            if(path) this_cookie+=";path="+path;
            document.cookie=this_cookie;
       }
       //根据key值，清除对应cookie
       this.removeItem=function(key){
            if(!(key in cookie)){
                return;
            }
            delete cookie[key];
            this.length--;
            for(var i=0;i<keys.length;i++){
                if(keys[i]===key){
                    keys.splice(i,1);
                    break;
                }
            }
            document.cookie=key + "=;max-age=0";
       }
       //清除所有cookie
       this.clear=function(){
            for(var i=0;i<keys.length;i++){
                document.cookie=keys[i]+"=;max-age=0";
            }
            cookie={};
            keys=[];
            this.length=0;
       }
       //返回所有的cookie
       this.geteAllCookie=function(){
          return cookie;
       }
    }

给个简单使用：

var mycookie=new cookieStorage(3*24*60*60,"/");
mycookie.setItem("name","wsp ");
mycookie.setItem("domain","wangshiping.net");
console.log(mycookie.getItem("name"));
mycookie.clear();
console.log(document.cookie);
