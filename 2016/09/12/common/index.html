什么是common.js

JavaScript是一个强大的面向对象语言，它有很多快速高效的解释器。官方JavaScript标准定义的API是为了构建基于浏览器的应用程序。然而，并没有定于一个用于更广泛的应用程序的标准库。

CommonJS API定义很多普通应用程序（主要指非浏览器的应用）使用的API，从而填补了这个空白。它的终极目标是提供一个类似Python，Ruby和Java标准库。这样的话，开发者可以使用CommonJS API编写应用程序，然后这些应用可以运行在不同的JavaScript解释器和不同的主机环境中。在兼容CommonJS的系统中。你可以使用JavaScript程序开发：

  • 服务器端JavaScript应用程序
  • 命令行工具
  • 图形界面应用程序
  • 混合应用程序（如，Titanium或Adobe AIR）

Nodejs 和Commonjs之间的关系

CommonJS是一种规范，NodeJS是这种规范的实现。

#coomonjs 的模块定义

commonjs 的模块定义分为 模块定义（exports）、模块引用(require)、模块标识(module)

  • require（） 用来引入外部模块、
  • exports 用来定义导出当前模块的变量或者方法、
  • module对象就代表模块本身。

使用一下模块：

//sum.js
exports.sum=function(a,b){
    return a+b;
}
//caculate.js
var math=require("sum.js");
var result=math.sum(1,3);

AMD 与common.js

因为commonjs 是为后端表现定义的规范、他是不适合前端的。于是乎，前端的异步模块定义出现了。AMD就是其中之一。 AMD定义的接口只有一个：

define(id?,dependencies?,factory)

他要在声明模块的时候，就定义好依赖，并且以形参的形式传递到factory中，比如这样：

define([dep1,dep2],function(dep1,dep2){
     //do something
});

如果没有依赖 就直接这样写：

define(function(){
    //do something 
    var exports={};
    exports.method=function(){
    }
    return exports
});

可以看到，AMD规范是封装在 define 这个函数中的。使用的 require.js 就是AMD规范的实现。

CMD 规范

sea.js 是国内的玉伯写的，并且提出了 CMD 规范，与AMD 相近，不过更加的好用一些：

define(function(require,exports,module){
    module.exports=function(){
        //do something...
    } 
});
