javascript节流函数的使用

发表于 2016-10-12   |   分类于 技术   |     |   阅读次数

什么时候用到了节流函数？

有这样一个需求啊，就是当浏览器改变大小的时候，需要改改页面上的元素大小、这个你怎么做？很自然的想到了监听浏览器窗口的 resize 事件、于是我们写下以下代码：

var n=0;   
function resizeHandle(){
    console.log(new Date().getTime());
    console.log(++n);
}
window.onresize=resizeHandle;

然后我们改变浏览器的窗口大小，发现打印信息如下：

可以看到，打印的了很多次，这显然不是我们所需要的，事实上，真正的resizeHandle 可能会有 ajax请求等，这样对性能就会有很大影响。

节流函数

思路很重要，我们只需要在窗口变化后执行一次，而事件的触发又不是我们能控制得了的，故我们采用延时调用的方式、在重复调用时，取消执行延时函数，直到最后一次，延时函数才会触发，这样便实现了，在过程中，不会出现重复调用。

function throttle(method,context){
    clearTimeout(method.methodId);
    method.methodId=setTimeout(function(){
        method.call(context);
    },500);
}
window.onresize=function(){
    throttle(resizeHandle,window);
}

再次改变窗口大小，如下图：发现resizeHandle函数只在最后停止改变窗口大小时执行一次，这样就避免了函数的重复调用。

好，我们看看另外一种网上的写法：

function throttle2(method,delay){
    var timer=null;
    var context=this, args=arguments;
    return function(){
        clearTimeout(timer);
        timer=setTimeout(function(){
            method.apply(this,args);
        },delay);
    }
}
window.onresize=throttle2(resizeHandle,500);

对比一下两种写法：

第一种是直接在调用方法上设置一个变量，通过这个变量来控制，是否要执行方法。第二个是返回了一个闭包，闭包引用事件函数的内的变量。

改一下需求

加入现在有个类似百度搜索的输入框，同样我们监听键盘的keyup 事件，如果每次起来都搜索显然，会太耗性能，于是，我们可以参考节流函数的思路，用定时函数来处理，但实际情况下，如果用户打字速度很快，我们在输入的过程中是不会出现搜索结果的，只有再输入结束的时候才出现。这样好像也与需求不一致。。于是我们可以做一下这样的改动，让输入框的监听事件，强制间隔一段时间执行。。来我们实现这样的节流函数。

//html 
输入项：<input type="text" style="width:200px;height:30px" id="inputTest">
//js
function throttle3(method,delay,timeLength){
    var timer=null;
    var context=this, args=arguments;
    var start=new Date().getTime();
    return function(){
        current=new Date().getTime();
        if(current-start>=timeLength){   // 如果时间间隔已经大于了设置的时间间隔，那么强制执行方法
            method.apply(context,args);
            start=current;
        }else{
            clearTimeout(timer);
            timer=setTimeout(function(){
                method.apply(context,args);
            },delay);
        }
    }
}
var input=document.getElementById("inputTest");
input.onkeyup=throttle3(resizeHandle,500,500);

运行一下，可以看到，一直敲击键盘,可以看到控制台会定时的调用打印方法，如果不加上定时打印，键盘敲击比较快，就只会在敲击结束后才打印内容。

#函数节流的应用场景

  • dom元素的拖拽功能的实现（mousemove）
  • 射击游戏的mousedown/keyup事件（单位时间内只能发射一颗子弹）
  • 计算鼠标移动的距离
  • 搜索联想
  • 监听滚动事件判断是否到页面底部，加载更多

总结一下

函数节流产生的原因一般就是事件的监听函数的触发太过频繁，而只需要在间隔一段时间才响应事件。故掌握了节流函数的思路。类似的应用场景也就不难应付了。
